Moving Highland Cows to Astro

Nick Jacobs | Saturday, February 5, 2022

Astro.build

Recently, one of my colleagues shared a link to astro.build on an internal Slack channel. I was intrigued by it for a number of reasons. I had been looking for a replacement for my Jekyll-based content for a while but had never really gotten excited about the other solutions I had tried along the way. I was hacking the pages by writing them in iA Writer and then using it to output HTML which I would copy and style by hand into the site. If that sounds awkward, it was. As a result my output really suffered.

I wanted something that was light and easy to use and Astro sure looked like it was checking those boxes. Astro provided a couple of things that I liked:

  1. There was HTML/CSS/JavaScript content for it but that was basically all framework code
  2. I could write my content purely in Markdown
  3. Styling seemed to be straightforward with a ReactJS-like approach.

After reading through the docs and noticing they had a straightforward way to host the content on github.io, I started thinking: “This could work…” The feature that sealed the deal was a theme called “Ink” that was prominently listed on the “Themes” page in the docs.

Astro - Ink

This Astro theme is written by Aftab Alam is based on a popular theme for Hugo also called “Ink”. The code is all available at github.com/one-aalam/astro-ink. I pulled that code and started hacking through it. I write “hacking” but the reality is that his code is incredibly well-written and even given the level of my JavaScript skills, I was able to easily modify it to suit my purposes.

Final Thoughts

The initial version of the site took me a few nights to put together and much of that was getting my head wrapped around how to organize things on GitHub so that publishing would be straightforward going forward.

Is this the framework for you? Perhaps. If I’ve learned anything it’s that everyone has their own preferences for these kinds of tools. I tried major sites such as Medium and Wordpress over the years but never really enjoyed their toolsets or potential barriers to access to my site. I think the one thing I miss is that I don’t have a way for people to comment on my blogs. Although Astro.build is primarily a static web page builder, there is no inherent reason it couldn’t be hosted in a web server that provides the necessary hooks to support such a capability.

Stay tuned :)
